# Cup Stacking Evaluation Alignment

The cup-stacking generation and evaluation environments load the same kitchen
scene and table asset. The visible mismatch comes from the cup start-position
distribution, not from a different table size.

| Object | Evaluation x range | Evaluation y range |
|---|---:|---:|
| `blue_cup` | `0.31` to `0.41` | `-0.45` to `-0.35` |
| `pink_cup` | `0.41` to `0.51` | `-0.45` to `-0.35` |

The original generation defaults use a different workspace and may also flip
the blue/pink relative layout. Use the evaluation-aligned profile to train on
the fixed evaluation distribution without modifying `eval/cup_stacking_eval.py`.

## Generate Evaluation-Aligned Data

Run inside the Isaac Lab Docker container. Use a new dataset repo instead of
mixing aligned demonstrations into an older dataset.

```bash
env -u DISPLAY \
VK_ICD_FILENAMES=/usr/share/vulkan/icd.d/nvidia_icd.json \
python scripts/datagen/generate.py \
  --task HCIS-CupStacking-SingleArm-v0 \
  --num_envs 1 \
  --device cuda \
  --headless \
  --enable_cameras \
  --record \
  --use_lerobot_recorder \
  --lerobot_dataset_repo_id MidoriChou/ai_final_7_eval_aligned \
  --object_poses data/ai_final_7/object_poses.json \
  --cup_layout_profile eval \
  --target_demo_count 500
```

`--cup_layout_profile eval` samples the two cup positions independently from
the same initial positions and `+/- 5 cm` x/y offsets used by evaluation. It
intentionally ignores the source cup positions, workspace clamp, pair flipping,
and minimum/maximum cup-distance settings.

Only successful state-machine episodes are exported. Start with 500 successful
episodes; increase toward 1000 if training and generation time allow.

Upload the completed dataset from inside the container:

```bash
hf upload MidoriChou/ai_final_7_eval_aligned \
  ~/.cache/huggingface/lerobot/MidoriChou/ai_final_7_eval_aligned/ \
  --repo-type dataset
```

## Train Candidate Models

Run training on the host. Do not package the model from the lowest training
loss alone; train candidates, run the fixed evaluation, then package the one
with the highest success count.

ACT is the recommended first candidate for this dataset because the
demonstrations are consistent, single-task, and generated by one scripted
trajectory family. The training helper changes ACT from its default 100
open-loop actions to per-step replanning with temporal ensembling.

```bash
scripts/train_cup_stacking_policy.sh \
  act \
  MidoriChou/ai_final_7_eval_aligned \
  outputs/train/cup_eval_act_seed42
```

Use Diffusion as the second comparison after the aligned dataset is working.
The helper uses a smaller network suitable for an 8 GB GPU and DDIM inference
to avoid excessively slow evaluation.

```bash
scripts/train_cup_stacking_policy.sh \
  diffusion \
  MidoriChou/ai_final_7_eval_aligned \
  outputs/train/cup_eval_diffusion_seed42
```

The helper defaults to 200,000 steps, batch size 4, AMP, moderate image
augmentation, and checkpoints every 20,000 steps. Override settings with
environment variables:

```bash
BATCH_SIZE=2 STEPS=300000 WANDB_ENABLE=true \
  scripts/train_cup_stacking_policy.sh act MidoriChou/ai_final_7_eval_aligned
```

## Select the Best Checkpoint

Use the fixed evaluation config unchanged:

```bash
python scripts/rollout.py \
  --task=eval/cup_stacking_eval.py \
  --policy_type=lerobot-act \
  --policy_checkpoint_path=outputs/train/cup_eval_act_seed42/checkpoints/last/pretrained_model \
  --policy_action_horizon=1 \
  --device=cuda \
  --enable_cameras \
  --seed=42 \
  --eval_rounds=50 \
  --episode_length_s=20
```

Evaluate the last checkpoint and earlier saved checkpoints. Training loss can
continue decreasing after simulation success has peaked. Compare at least
`100000`, `140000`, `180000`, and `last`, using the same 50 evaluation rounds
for every candidate.

## Create the Submission Zip

After choosing the best evaluated checkpoint:

```bash
python scripts/package_policy.py \
  outputs/train/cup_eval_act_seed42/checkpoints/180000/pretrained_model
```

This validates the model, config, preprocessing, postprocessing, and
normalization files, then creates:

```text
checkpoints/my_policy/
checkpoints/my_policy.zip
```

Upload `checkpoints/my_policy.zip`. The zip contains `my_policy/` as its root
directory and includes all files required for inference.
